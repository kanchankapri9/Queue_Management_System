Renaming a branch to "main" is often done to align with modern naming conventions and to promote inclusivity .Renaming

which has been the traditional default name for the primary branch in Git repositories. 


question :

but i checked by git branch it was showing "main" already, so why would we rename it again?
   
answer :

Renaming a branch to "main" is often done to align with modern naming conventions and to promote inclusivity.
yes you have checked and found that the branch is already named "main," so there is no need to rename it again. 
The command `git branch -M main` is typically used when you want to rename an existing branch to "main," but if it's already named "main," running the command again won't have any effect. 
It's just a precautionary step to ensure that the branch name is consistent with the desired naming convention.
